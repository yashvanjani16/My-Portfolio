<a> tag have tab focus by default, if we want tab focus on div or p or any other tag than put tab-index=0,if we dont want tab focus than put tabindex=-1
tabindex="-1" will remove tab focus from the element, <a> should have '[href="#"]' in order to work "focus-css" 


If any element consists role="heading" ,then aria-level="1" should be present. To set globally using JS -- [$j('.banner-content p[role="heading"]').attr('aria-level','1');]
role="presentation" should be present in thead tag,if no thead then in table tag.

var allTabLists = document.querySelectorAll('ul[role="tablist"]');

// Loop through each <ul> element
allTabLists.forEach(tabList => {
    // Find all <li> elements within the current <ul>
    var tabLis = tabList.querySelectorAll("li");
    
    // Loop through each <li> element and set its role attribute to "tab"
    tabLis.forEach(tab => {
        tab.setAttribute('role', 'tab');
    });
});

//Adding aria-label attribute
Example 1: Using legacyBehavior (Next.js 12 and earlier)

<Link href="/about" legacyBehavior>
  <a aria-label="Go to About page">About</a>
</Link>

Example 2: Directly on Link (Next.js 13+)
In Next.js 13+, <Link> automatically renders an <a> tag, so you don't need to wrap it

<Link href="/about" aria-label="Go to About page">
  About
</Link>
